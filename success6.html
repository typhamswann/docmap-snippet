Authentication

Our authentication system uses OAuth 2.0 and JSON Web Tokens (JWT).

  1. POST /api/auth/login
    Request body: { "username": "string", "password": "string" }
    Response: { "token": "", "expires": 3600 }
  2. POST /api/auth/logout
    Invalidates the current token.

Include your token in the Authorization header on protected routes: Authorization: Bearer <jwt-token>

Endpoints

Below are the main REST endpoints grouped by resource. All endpoints require an Authorization header if they access protected data.

  • GET /api/users/:id
  • PUT /api/users/:id
  • DELETE /api/users/:id

  • DELETE /api/products/:id

Orders

Usage Examples

  1. Creating a New User (POST /api/users)
  2. Logging In (POST /api/auth/login)
  3. Fetching Protected Resource (GET /api/users)
  4. Creating an Order (POST /api/orders)
  5. Logging Out (POST /api/auth/logout)

Error Handling

The API returns standard HTTP status codes, e.g. 400, 401, 403, 404, 409, 500.

{
  "error": "Invalid credentials",
  "code": 401
}

Filtering & Querying

Many endpoints accept query parameters for filtering results, e.g. ?status=active&role=admin or ?price_lt=1000.

Bulk Operations

  • POST /api/users/bulk-create
  • PUT /api/users/bulk-update
  • DELETE /api/users/bulk-delete

Webhooks

  • GET /api/webhooks
  • POST /api/webhooks
  • DELETE /api/webhooks/:id

Events include: userRegistered, userDeleted, orderCreated, orderUpdated, etc.

Rate Limits

100 requests/minute and 1000/day per token. Exceeding results in 429 Too Many Requests.

Websockets

Real-time updates at wss://api.example.com/realtime, passing ?token=<jwt>.

SDKs & Libraries

Official: JavaScript/TypeScript, Python, Java. Community: PHP, Ruby, etc.

Permissions

Role-based access control (RBAC): admin, manager, user, guest. Permission scopes: user:read, product:update, order:delete, etc.
